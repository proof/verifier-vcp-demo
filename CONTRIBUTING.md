# Contributing

## Requirements

- Node.js
- npm

## Getting started

See the [README](./README.md#getting-started) for setup instructions.

## Bug fixes

If you've found a bug you'd like to fix, open a pull request with a clear description of the problem and how your changes address it.

## New features

If you have a feature idea, open a GitHub Discussion before starting work on it. The scope of the demo app is limited, so we want to make sure new features align with the project's direction before any significant effort is invested.

## Coding standards

Code formatting is enforced with Prettier (including Tailwind class sorting). Check your code against the project's standards by running:

```sh
npm run lint
```

To fix formatting violations:

```sh
npm run format
```

## Pull request process

When submitting a pull request:

- Make sure your branch is up to date with `main`
- Include a clear title and description explaining what changed and why
- Keep changes focused, try to limit one issue or feature per PR

## Code of conduct

This project follows the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). By participating, you are expected to uphold this standard.

## Communication

- **GitHub Issues** — bug reports
- **GitHub Discussions** — feature ideas and questions
- **GitHub Pull Requests** — code contributions
